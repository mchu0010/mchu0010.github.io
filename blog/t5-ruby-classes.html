Classes and Instance Variable

Bluegills Chicago - Phase 0 Week 5

October 19, 2014

CLASS
What is class? In ruby, a class is a way of organizing and producing objects with similar attributes and methods.
A basic class consists of the word class and the name of your class, in this case "Lakers".

The program above shows an example of a class called Lakers. What this code does is when you input the player’s name, position,
and the years pro, it will output all that information with the player’s draft class. There are three methods in this that can
be called on.

class Lakers
end


INITIALIZE
Under "class Lakers", you can see that I defined an initialize method. What this "def initialize" method does is it makes
this method execute every time a a new instance of class is created.

def initialize(name, position, years_pro)
  @name = name
  @position = position
  @years_pro = years_pro
end


INSTANCE VARIABLES
Under the "def initialize", there variables with the "@" sign in front of it. These are called instance variables and the the
difference between local variables and instance variable is that an instance variable retains the value assigned to it even
after a method in which it was initialized has terminated. This means that the variable is attached to the instance of the
class. For example, in the program below player has his own @name ("Kobe Bryant"), @position ("Shooting Guard"), and a
@years_pro (18). Those variables belong to the player instance, which is why they’re called instance variables.

@name = name
@position = position
@years_pro = years_pro


NEW OBJECT
We can create an instance of a class just by calling ".new" on the class name. At bottom of the entire Lakers class, we defined
a new object called "player = Lakers.new" with the three argument it’s 3 arguments: name, position, and years_pro. Also, below the
new object, you can see that we called 2 methods on our object kobe and that outputs:

Player: Kobe Bryant
Position: Shooting Guard
Draft Class: 1996
