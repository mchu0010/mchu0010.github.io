Ruby vs JavaScript

Bluegills Chicago - Phase 0 Week 7

November 2, 2014

After beginning JavaScript this past week, I realized there are some similarities between the two object oriented languages. But so far, with my limited use of JavaScript, I definitely prefer using Ruby for many reasons. JavaScript has more flexibility in terms of coding, and it gives you a variety of ways to code depending on your preferences. For example, from the Eloquent JavaScript link we read from, it said that sometimes a " ; " is required but the code can still run even without. However, with that flexibility, it can also lead to numerous and messy errors in the code. On the other hand, Ruby is a lot stricter and the errors are more specific so that we know what is wrong with the program.

Ruby Hashes Vs JavaScript Objects

RUBY

In Ruby, hashes are a way of storing data using a collection of key-value pairs. Values are assigned to keys using the "=>" sign. Hashes can be accessed by calling on the key that corresponds to each value.

hash = {
key1 => value1,
key2 => value2,
key3 => value3
}

hash["key1"]
## accessing "key1" to get the value of "value1"

JAVASCRIPT

In JavaScript, objects are very similar to Ruby's hashes. The keys are called properties and instead of using " => ", we use " : " to separate the keys and keys and values. These values can be accessed by typing the object name and the name of the property with a period in between the two, just like calling a method in Ruby.

var object = {
property1: "value1"
property2: "value2"
property2: "value3"
}

object.property1
// accessing "property1" to get the value of "value1"
